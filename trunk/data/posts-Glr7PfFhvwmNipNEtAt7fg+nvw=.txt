      "body": "<p>The expression <code>p++</code> which you've written is at namespace scope.  It is forbidden by the grammer of <em>namespace-body</em> which is defined in §7.3.1/1 as:</p>\n\n<blockquote>\n  <p>namespace-body:<br>\n  &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;declaration-seq<sub>opt</sub></p>\n</blockquote>\n\n<p>which says the namespace-body can <em>optionally</em> contain <strong>only</strong> <em>declaration</em>. And <code>p++</code> is surely not a declaration, it is an expression, therefore the Standard implicitly forbids it. The Standard might have explicit statement forbidding this, but I think the above should be enough.</p>\n\n<p>In the same way, you can write these at namespace scope:</p>\n\n<pre><code>namespace sample\n{\n  f(10,10); //error\n  std::cout &lt;&lt; \"hello world\" &lt;&lt; std::endl;//error\n}\n</code></pre>\n\n<p>But if you somewhow convert <em>expressions</em> into <em>declarations</em>, then you could evaluate the so-called expressions. Here is one trick:</p>\n\n<pre><code>#include&lt;iostream&gt;\n\nnamespace sample\n{\n  struct any { template&lt;typename T&gt; any(const T&amp;){} };\n\n  void f(int a,int b) { std::cout &lt;&lt; a * b &lt;&lt;  std::endl; }\n\n  any a1= (f(10,10), 0); //ok\n  any a2 = std::cout &lt;&lt; \"hello world\" &lt;&lt; std::endl;//ok\n}\n\nint main() {}\n</code></pre>\n\n<p>Output (if you're lucky):</p>\n\n<pre><code>100\nhello world\n</code></pre>\n\n<p>Online demo : <a href=\"http://ideone.com/icbhh\" rel=\"nofollow\">http://ideone.com/icbhh</a></p>\n\n<p>Notice that the return type of <code>f()</code> is <code>void</code>, which means I cannot write the following (<a href=\"http://ideone.com/YmGpg\" rel=\"nofollow\">see error)</a>:</p>\n\n<pre><code>any a1 = f(10,10); //error\n</code></pre>\n\n<p>That is why I used <em>comma</em> operator so that every expression could have some value, which evaluates to the last operand in the comma expression. The answer is, I wrote this so that I could assign value of <em>any</em> type (no pun intended), be it <code>int</code>, <code>std::ostream&amp;</code> or whatever.
